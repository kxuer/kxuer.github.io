# flex

# 注意事项

  1. 字体大小会影响flex-basis: 75%;其实际占的比例可能就不是75%。 => 思考：css优先级。

  2. box-sizing: content-box|border-box => 思考：|解决浏览器兼容吗？

box-sizing: border-box; 可能会影响最终的高度，进而是否出现scroll滚动条。

  1. 有时候肉眼看到的不一定就是那样，放大后发现真的是垂直居中。

  2. max-width会影响text-overflow: ellipsis;的作用，貌似max-width的优先级别高。 => 思考：css优先级。

text-overflow: ellipsis;至少还需要： width：value；overflow: hidden; white-space: nowrap;，而且overflow: hidden不能从父元素中继承。 => 思考：html父子元素的css继承。

  1. 浮动清除clear：如果声明为左边或右边清除，会使元素的上外边框边界刚好在该边上浮动元素的下外边距边界之下。

  2. 默认无边框，如果希望边框出现，就必须声明一个边框样式border-style。

  3. display:block - 把链接显示为块元素可使整个链接区域可点击（不仅仅是文本），同时也允许我们规定宽度。

  6. 只设置一个元素的margin: 0;而不设置该元素的兄弟元素的margin: 0;可能会影响到父元素的。

  7. li的子元素div设置width:100%;无效？为什么？当父子元素都是块级元素，子元素宽度width是父元素的宽度减去子元素的margin值和padding值。 => 思考：html父子元素的css继承。

  9. 滚动条：css修改滚动条默认样式 (opens new window)

<div style="height: 118px; overflow: hidden; ">
    <div class="innerbox" style="height: 100%; overflow-y: auto;"></div>
</div>

    border-radius: 5px;
    -webkit-box-shadow: inset 0 0 5px rgba(0,0,0,0.1);
    background: rgba(0,0,0,0.1);
}
.innerbox::-webkit-scrollbar-track {/*滚动条里面轨道*/

    border-radius: 0;
    background: rgba(0,0,0,0.1);
}
